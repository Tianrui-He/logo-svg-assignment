# Gemma 3 270M SVG 徽标生成 LoRA 实验报告

## 1. 任务与结论概览

本项目在 219 条“详细提示词 → SVG 徽标”训练样本上对
`google/gemma-3-270m` 进行 LoRA 微调，并在固定的 17 条公开验证集上比较同一基座模型
目的是验证小模型相对自身基线是否
提升，并分析自定义代理 reward 与真实视觉质量之间的一致性。

最终 LoRA 在自定义 reward 上从 **1.8824** 提升到 **22.8096**，平均提升
**20.9272**；可解析 SVG 比例从 **0%** 提升到 **23.53%（4/17）**。因此模型确实学到
了部分 SVG 结构，但中位数仍为 2 分，且高分样本存在明显的语义错配和重复路径。结论是：
**结构有效性有小幅、可测量的相对提升，但还没有形成稳定的提示词到图形语义映射。**

## 2. Reward 设计

代理指标实现在 `student_kit/reward.py`，返回 0–100 分及可解释的子项：

| 子项 | 权重 | 设计理由 |
|---|---:|---|
| Validity | 30 | 检查单一完整 SVG 根、XML 可解析性、`viewBox`、危险脚本/外部引用等；不能可靠渲染的结果首先应被判为失败。 |
| Geometry | 20 | 检查退化几何、极端/越界坐标、悬空局部引用和有效可见比例。 |
| Composition | 15 | 对可见元素数量、颜色数量、标签多样性和画布利用率使用软区间，避免空白或无限膨胀的结构。 |
| Prompt fidelity | 25 | 检查提示中的显式/命名颜色、基础形状词、渐变或扁平风格以及简单度要求。 |
| Anti-degeneration | 10 | 惩罚纯背景、完全重复路径、异常长度、不可见结构等退化输出。 |

出现脚本、事件属性或外部 URL 时，总分封顶为 20。各项采用软阈值，因为合法徽标可以合理
使用开放路径、重复圆点或覆盖画布的背景矩形。

### 2.1 Reward 校准

在 236 个目标 SVG 与人工构造的负例上进行校准：目标参考均分 **98.12**、中位数
**98.85**、最低 **90.16**；破损 XML 为 **2.00**，空 SVG 为 **42.15**，纯背景为
**59.82**，脚本注入被安全规则封顶为 **20.00**。7 个自动化测试覆盖 XML、退化、坐标、
安全和正常参考样本。

这只能证明 reward 对训练分布中的合法结构不过度误杀，不能证明它理解任意 `<path>` 画出
的对象。后面的高分失败案例验证了这一局限。

## 3. 数据、环境与训练设置

- 数据：219 条训练样本，17 条固定验证样本；训练日志中的 token 长度为
  `1326.57 ± 506.84`，最大 3534，均未超过 `max_length=4096`。
- 基座：ModelScope `google/gemma-3-270m`。
- 环境：百度 AI Studio DSW-GPU，Ubuntu 22.04，Python 3.12，ms-swift 4.2.3，
  Transformers 5.8.1，PEFT 0.19.1；训练使用单 GPU、BF16。日志未保留具体 GPU 型号。
- LoRA：rank 16、alpha 32、dropout 0.05，目标为全部线性投影层；可训练参数
  3.797M，占总参数 1.3965%。
- 优化：学习率 2e-4，cosine 调度，weight decay 0.01；单卡 batch 1，梯度累积 8，
  有效 batch 8；最多 8 epoch，只在 assistant SVG 上计算损失。
- 训练耗时：449.6 秒；日志报告峰值显存约 21.76 GiB。
- 自评解码：`do_sample=false`、`num_beams=1`、`max_new_tokens=2048`、
  `repetition_penalty=1.0`、seed 42；基座和 LoRA 使用完全相同设置。

完整参数见 `train_config.yaml`，原始训练证据见 `train_E1.log`。

## 4. 实验与 checkpoint 选择

本次完成了基座 E0 与 rank-16 配置 E1。没有把未运行的 rank/LR 变体伪装成实验结果；
有限算力优先用于完成固定 17 条的公平生成评测。E1 每个 epoch 均保存并验证，可用于观察
早停趋势：

| Epoch / step | Eval loss | Eval token accuracy |
|---:|---:|---:|
| 1 / 28 | 0.82610 | 0.72749 |
| 2 / 56 | 0.73839 | 0.74930 |
| 3 / 84 | 0.70136 | 0.76035 |
| 4 / 112 | 0.68082 | 0.76494 |
| 5 / 140 | 0.67203 | 0.76735 |
| 6 / 168 | 0.66903 | 0.77033 |
| 7 / 196 | 0.66801 | 0.77195 |
| 8 / 224 | **0.66729** | **0.77257** |

验证损失没有反弹，故按预先指定的 `metric_for_best_model: loss` 选择
`checkpoint-224`。这里存在一个限制：只对最终 checkpoint 完成了全量生成 reward 评测，
所以不能声称它同时也是“生成 reward 最优”的 checkpoint。

## 5. 基座与 LoRA 自评结果

以下数字直接来自 `results.json`：

| 指标 | 基座 270M | LoRA | Δ |
|---|---:|---:|---:|
| Mean total reward | 1.8824 | 22.8096 | +20.9272 |
| Median total reward | 2.0000 | 2.0000 | 0.0000 |
| Valid SVG rate | 0.00% | 23.53% | +23.53 pp |
| Validity | 1.8824 | 8.8235 | +6.9411 |
| Geometry | 0.0000 | 4.7059 | +4.7059 |
| Composition | 0.0000 | 3.1961 | +3.1961 |
| Prompt fidelity | 0.0000 | 3.8367 | +3.8367 |
| Anti-degeneration | 0.0000 | 2.2474 | +2.2474 |

平均分和有效率均提升，说明 LoRA 至少学会了以 SVG 语法组织输出，并在 4 个样本中成功
结束文档。但 13/17 的 LoRA 输出仍未闭合，常在重复路径后达到 2048-token 生成上限。
因此不能把 +20.93 简单解释为整体视觉质量提升；中位数不变更能反映多数样本仍然失败。

## 6. 定性案例

### 6.1 最大代理分提升：case 7

该提示要求深色圆徽章、白色厨师帽、橙红火焰、交叉白色餐叉和白色外圈。基座输出为
不完整文本，得 2.0；LoRA 输出完整可解析 SVG，得 **98.379（Δ +96.379）**。

- 基座：[best_delta_07_base.svg](examples/best_delta_07_base.svg)
- LoRA：[best_delta_07_adapted.svg](examples/best_delta_07_adapted.svg)

从结构角度这是明确提升，但人工查看发现 LoRA 主要画了重复的橙色曲线，没有可辨认的
厨师帽和餐叉。它满足了颜色、圆形、渐变和 XML 结构等代理特征，却没有满足对象语义，
因此属于“结构提升、视觉保真度未同步提升”。

### 6.2 无明显变化：case 13

基座与 LoRA 都只得到 2.0。LoRA 虽以 `<svg>` 开始，但在重复生成后没有输出闭合标签，
因此仍无法作为图像渲染。这代表验证集中的多数情况。

- 基座：[median_delta_13_base.svg](examples/median_delta_13_base.svg)
- LoRA：[median_delta_13_adapted.svg](examples/median_delta_13_adapted.svg)

### 6.3 最差并列案例：case 2

该案例同样为 2.0 → 2.0。LoRA 从自然语言退化转为不完整 SVG，但没有跨过“完整可解析”

## 7. 失败分析与 Goodhart 效应

1. **重复和截断仍是主要失败模式。** 17 条 LoRA 输出全部以 SVG 开始，但 13 条没有
   `</svg>`；这些不完整输出长度集中在约 2500–3340 字符，并包含明显重复路径，符合生成
   到 token 上限的表现。
2. **出现模板/颜色坍缩。** 13/17 输出复用超大背景矩形模板，10/17 使用
   `#FBF7E3`，即使提示要求海军蓝、青色、红色等其他配色。
3. **高 reward 不等于高视觉保真度。** case 6 要求六边形蜂巢、蜂蜜和蜜蜂，却主要生成
   重复浅色水滴路径，仍得 88.210；case 11 要求太阳、海浪和卡通鱼，却主要生成浅色圆和
   退化弧线，仍得 84.794。
4. **代理为何被钻空子。** Reward 可以可靠检查 XML、颜色字符串、元素数量、重复率与
   坐标，但无法从任意 path 数据判断它是否真的像“蜜蜂”“厨师帽”或“鱼”。因此模型只要
   组合出合法结构和提示中提到的若干颜色/形状词，就可能得到过高的 prompt-fidelity 分数。
5. **冻结评测风险。** 老师的私有 Sonnet 视觉评审能够识别上述语义错配，所以公开代理的
   +20.93 很可能高估真实提升。不过有效 SVG 比例从 0 到 23.53% 是较可信的结构性进步。

## 8. 可复现性与局限

仓库包含 LoRA adapter、精确训练配置、训练日志、固定解码的逐样本 `results.json`、reward
实现、校准结果和回归测试。`adapter/adapter_model.safetensors` 与所选
`checkpoint-224` 的 SHA256 一致。随机种子固定为 42；不同 GPU 内核和库版本仍可能产生
小幅差异。

本实验没有完成 rank 8 或学习率 1e-4 的额外训练，也没有逐 checkpoint 比较生成 reward；
这限制了对最优超参数的结论。

## 9. 结论

LoRA 相对未微调 270M 基座取得了可复现的小幅结构提升：平均代理 reward +20.9272，
有效 SVG 率 +23.53 个百分点。但中位数仍为 2，且高分样本暴露出严重 Goodhart 效应，
所以不能宣称模型已经学会高保真的徽标绘制。这个结果符合极小模型任务的预期，也说明
reward 的强项是可解析性与安全检查，弱项是任意向量路径的对象语义判断。
